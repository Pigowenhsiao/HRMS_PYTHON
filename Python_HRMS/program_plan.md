# Python HRMS 開發計畫（SQLite 版）

## 階段與里程碑
1) 基礎環境：建立虛擬環境；安裝 PyQt5、SQLite 驅動（標準庫 sqlite3）、pydantic 或 dataclasses、logging；建立專案結構（ui/, services/, dao/, models/, tests/, data/）。
2) 資料庫與設定：
   - 定義 SQLite schema（員工、個資、學歷、證照、訓練紀錄、證照-機台、權限、部門/區域/職務字典）。
   - 提供 Access → SQLite 匯入腳本與資料驗證（筆數、欄位對應、必填）。
   - 建立 config.json/.env 管理 DB 路徑、匯出路徑、版本號、權限選項。
3) UI 骨架：
   - 重構主畫面（已優化）；為各模組建立視窗/對話框（員工、個資、學歷、證照設定、訓練紀錄、對應、權限、報表、到期檢核）。
   - 統一中文字型/編碼，設定間距與群組版面；導入多語詞條並驗證切換。
4) 功能迭代：
   - v0.1：員工基本資料/個資/學歷 CRUD + 查詢；多語詞條基礎覆蓋主畫面/員工模組。
   - v0.2：證照設定、訓練紀錄 CRUD、證照-機台對應；CSV 匯出（失敗落地本機）；多語詞條擴充至證照/對應/匯出訊息。
   - v0.3：到期檢核、報表篩選/匯出；權限控管（顯示/鎖定）與版本檢查流程；報表/匯出多語完成。
5) 穩定化：
   - 單元測試（DAO/service），UI 冒煙測試；錯誤提示與日誌格式化。
   - 打包與啟動腳本（如 PyInstaller），附預設 SQLite DB 與範例設定；多語詞條審校。

## 技術選擇
- PyQt5 做 UI；sqlite3 做內建資料庫；pandas 可選用於 CSV；python-dotenv/configparser 處理設定；logging 處理日誌。
- DAO 層採參數化查詢；Service 層處理驗證、權限與匯出；UI 層只負責互動；多語詞條集中一份 JSON/YAML。

## 風險與緩解
- 資料遷移：以比對筆數/欄位、抽樣校驗降低遺漏；匯入腳本支援 rollback。
- 權限與安全：避免硬編碼帳號/路徑；透過設定檔+加鎖控件控制可操作項；預設 admin 密碼需上線前更換/雜湊。
- 匯出與路徑：若外部分享不可寫，提示並改寫本機；記錄錯誤以利重試；匯出訊息依語系切換。
- 編碼/字型：統一 UTF-8 與中文字型，避免亂碼；多語詞條需審校。
