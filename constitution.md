# Python HRMS 憲章

## 使命
以 PyQt5 + SQLite 建立桌面版 HRMS，完整複製既有 VB/Access 系統的員工、證照、權限與報表功能，並保留匯出至外部系統的能力。

## 設計原則
- 單一責任：UI 只處理互動，資料存取封裝於 service/dao；匯出/版本檢查獨立模組化。
- SQLite 優先：預設使用 `./data/hrms.db`，必要時以適配層切換 MySQL；不再依賴 Access/Jet。
- 資料遷移友善：提供 Access → SQLite 匯入腳本與校驗工具，保持欄位對應一致。
- 安全：輸入驗證、參數化查詢、分級權限；避免將帳號或路徑寫死於程式碼，使用設定檔。
- 可維護：type hints、logging、模組化結構、單元測試；錯誤訊息友善並可追蹤。
- 在地化/多語：UI、報表、匯出訊息皆使用 i18n 資源，預設支援繁中/英文/日文，集中管理詞條以便擴充與審校。
- 可移植：保持離線可用，匯出失敗需提示並改寫本機；不綁定特定網路分享路徑。

## 目前版號與重點
- 現行：`v0.3.1`（多語完整套用、登入權限與按鈕顯示控制、部門/區域/職務維護、證照/訓練紀錄避免重複新增、主畫面版本同步 config）。
- 歷程：`v0.1.x` 建立 SQLite schema、Access 匯入、員工/個資/學歷 CRUD；`v0.2.x` 證照/訓練/對應與 CSV；`v0.3.x` 報表、到期檢核、權限與多語完善。
